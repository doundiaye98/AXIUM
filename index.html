AXIUM — ce dossier est servi par WAMP, le site par Flask

Si vous voyez une erreur 404 du style « L’URL demandée est introuvable sur le serveur » en ouvrant une adresse du type /_axium/admin ou /services/ sous http://localhost/..., c’est normal : Apache ne connaît pas ces chemins. Il faut lancer l’application Python puis utiliser le port Flask (souvent 5001).

0. ERR_CONNECTION_REFUSED sur 127.0.0.1 = rien n’écoute (fenêtre Flask fermée, plantage au démarrage, ou mauvais numéro de port). Relancez DEMARRER-ICI.cmd ou Demarrer-AXIUM-port-5002.cmd, gardez la console ouverte, attendez « Running on… », puis utilisez le même port dans le navigateur.

1. Double-clic sur DEMARRER-ICI.cmd à la racine du projet (laissez la fenêtre ouverte).

2. Ouvrez dans le navigateur :

  • Accueil du site
  • Panneau maintenance (admin)

Si le port affiché au démarrage n’est pas 5001, remplacez 5001 dans ces liens.

404 en français sur 127.0.0.1:5001 ? Le port est souvent pris par un autre service (WAMP, etc.) et Flask n’a pas démarré. Test rapide : ouvrez /health — vous devez voir le texte ok seul. Sinon ce n’est pas AXIUM. Utilisez un autre port : double-clic sur Demarrer-AXIUM-port-5002.cmd à la racine du projet, ou dans PowerShell : $env:PORT='5002' puis relancez le démarrage.
